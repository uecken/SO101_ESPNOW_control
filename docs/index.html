🦾 SO-101 Leader-Follower

Web ブラウザから直接ファームウェアを書込めます。PlatformIO 不要。

準備

  1. Chrome / Edge (デスクトップ版) で本ページを開く
    • ❌ Safari / Firefox / モバイルブラウザは Web Serial API 非対応のため書込み不可
  2. 書込みたい MCU を PC に USB 接続
  3. 該当 board の 書込む ボタンを押す → シリアル port 選択 → 自動書込み

ボード選択

M5StickC ESP32

基板裏に TX=G32 / RX=G33 / GND / 5V を配線 (サーボバス側)。

⚠ このブラウザは Web Serial 非対応です。Chrome / Edge を使用してください。 ⚠ HTTPS 接続が必要です。

M5Stack AtomS3 ESP32-S3

TX=G2 / RX=G1 / GND / 5V をサーボバスへ。

⚠ 書込み前に boot mode に入れてください:
AtomS3 側面のボタンを 押したまま USB を PC に接続 → 接続後ボタンを離す。
(Arduino v2 + native USB-CDC の制約で auto-reset が効かないため)

⚠ Chrome / Edge を使用してください。 ⚠ HTTPS 接続が必要です。

Seeed XIAO ESP32-C3 ESP32-C3

TX=D6 (GPIO21) / RX=D7 (GPIO20) / GND / 5V をサーボバスへ。
Waveshare サーボドライバの場合 3-pin ケーブル直結で OK。

⚠ Chrome / Edge を使用してください。 ⚠ HTTPS 接続が必要です。

Seeed XIAO ESP32-S3 ESP32-S3

TX=GPIO43 / RX=GPIO44 / GND / 5V をサーボバスへ。

⚠ Chrome / Edge を使用してください。 ⚠ HTTPS 接続が必要です。

書込み後の初回設定 (device 毎に 1 回)

Leader と Follower それぞれに役割を設定 + 開始:

git clone https://github.com/uecken/SO101_ESPNOW_control
cd SO101_ESPNOW_control
pip install pyserial

# Leader (COMxx は USB 接続ポート番号)
python python/so101_setup.py --port COMxx --leader --start

# Follower
python python/so101_setup.py --port COMyy --follower --start

モード・サーボ ID・WiFi 設定等は NVS 永続化されるため、次回以降は 電源 ON のみで自動 RUNNING 復帰します (上記コマンドは書込み後 1 回だけ)。

サーボのキャリブレーションは別途 python/so101_calibrate.py --reset --port COMxx。 詳細は README 参照。
